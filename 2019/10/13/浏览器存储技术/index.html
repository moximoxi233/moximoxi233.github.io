浏览器存储技术

发表于 2019-10-13 | 分类于 web

常见的浏览器存储技术

cookie

cookie会随着每次HTTP请求头信息要一起发送，无形中增加了网络流量，且cookie存储的数据容量有限，会根据浏览器不同而决定(IE6只存储2K)

  • 缺点是：限制保存数据空间大小、数据保密性差、代码操纵复制等
创建cookie
  • 告诉浏览器 cookie 属于什么路径。默认情况下，cookie 属于当前页。
读取cookie
删除cookie
  • 删除 cookie 时不必指定 cookie 值
  • 直接把 expires 参数设置为过去的日期即可
  • 注意：定义 cookie 路径以确保删除正确的 cookie。如果你不指定路径，一些浏览器不会让你删除 cookie。
修改cookie
  • 创建 cookie 一样改变它，那么，旧 cookie 被覆盖。

web Storage

Web Storage分为会话数据与长期数据

  • sessionStorage(保存会话数据)，
  • localStorage(在客户端长期保存数据)。

可以将客户端数据分类型进行存储，使它在运用上优越于传统、单一的Cookie; Web Storage对比Cookie

web storage：

  1. 存储空间比cookie大
  2. 存储内容不会发送到服务器。而cookie的内容会随着请求一并发送到服务器，这对于本地存储是一种宽带浪费。
  3. 独立的存储空间。不会造成数据混乱。
  4. 更多丰富易用的接口，使数据操作更为简单
localStorage(在客户端长期保存数据)

localStorage是Web Storage互联网存储规范中的一部分,LocalStorage对数据存储没有时间限制，也就是说数据信息在网站关闭后依然存在，下次登录还可以看见

缺点：低版本浏览器不支持。

使用：

sessionStorage(保存会话数据)

用这种方式存储的数据仅窗口级别有效，同一个窗口（或者Tab）页面刷新或者跳转，都能获取到本地存储的数据，当新开窗口或者页面时，原来的数据就失效了。

  • sessionStorage仅在当前浏览器中有效，关闭浏览器数据消失

缺点：IE不支持、不能实现数据的持久保存。
使用：

sessionStorage、localStorage和cookie区别？

1.数据的有效性不同。

  • localStorage 关闭浏览器数据不会失效。
  • sessionStorage 仅在当前浏览器中有效，关闭浏览器数据失效。
  • cookie 只在设置cookie过期时间内有效，无论浏览器是否关闭。

2.作用域不同。

  • localStorage 在所有同源窗口中共享；
  • sessionStorage 不在不同浏览器窗口中共享。
  • cookie 也是所有同源窗口中共享。

userData

IE浏览器可以使用userData来存储数据，容量可达到640K，这种方案是很可靠的，不需要安装额外的插件。

​ :( 缺点：它仅在IE下有效。

Flash ShareObject

这种方式能能解决上面提到的cookie存储的两个弊端，而且能够跨浏览器，应该说是目前最好的本地存储方案。不过，需要在页面中插入一个Flash，当浏览器没有安装Flash控件时就不能用了。所幸的是，没有安装Flash的用户极少。

​ :( 缺点：需要安装Flash插件。

Google Gear

Google开发出的一种本地存储技术。

​ :( 缺点：需要安装Gear组件。

globalStorage

使用于Firefox2+的火狐浏览器，类似于IE的userData。

​ :( 缺点：IE不支持。
