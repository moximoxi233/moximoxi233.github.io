vue基础学习笔记

发表于 2019-10-24 | 分类于 vue

Vue 是一套用于构建用户界面的渐进式JavaScript框架。与其它大型框架不同的是，Vue 被设计为可以自底向上逐层应用。Vue 的核心库只关注视图层，方便与第三方库或既有项目整合。

  • 核心只关注视图（view）
  • 易学，轻量，灵活
  • 适用于移动端项目
  • 渐进式框架

介绍：

Vue.js 是一套构建用户页面的框架，只关注视图层 ，易上手，便于与第三方库或既有项目整合。Vue.js 、Angular.js 和 React.js 是前端三大主流框架，使用框架可以提高开发效率

vue中一个核心概念：让用户不再操作DOM元素，解放用户双手，让程序员可以更多的时间去关注业务

框架和库的区别

框架: 是一套完整的解决方案（如果需要更换框架，则需要重新架构整个项目）

  • node 中的express

库（插件）：提供某个小功能，对项目侵入性小，容易切换

  • 从jquery 切换到 zepto
  • 从 EJS 切换到 art-template

Node（后端）中MVC 与前端中 MVVM之间的区别

  • MVC 是后端的分层开发概念
  • MVVM 是前端图层概念，主要关注视图层

computed

计算属性的使用与特点

在 computed 中，我们可以定义一些属性，被叫做计算机属性。【计算属性】本质是一个方法，但我们使用这些计算机属性的时候，十八它们的名称，直接当作属性来使用

⚠

  • 【计算属性】在引用的时候，一定不要加（），直接当 属性 去使用
  • 其 function 内部，所用到的data中数据发生变化，就会重新计算该计算属性的值
  • 【计算属性】求值结果，会被缓存起来，方便下次直接使用，如果计算属性方法中没有任何数据发生变化，不会重现对计算属性求值

可以使用 methods 来替代 computed，效果上两个都是一样的，但是 computed 是基于它的依赖缓存，只有相关依赖发生改变时才会重新取值。而使用 methods ，在重新渲染的时候，函数总会重新调用执行。

组件（Component）是 Vue.js 最强大的功能之一。组件可以扩展 HTML 元素，封装可重用的代码。

vuex

目的：将一些公共数据存储到一个地方，方便修改和调用

main.js

只要将store挂载到vm实例中，人合租加纳都能使用store来存取

mutations

如果组件想要调用 mutations 中的方法，只能使用 –this.$store.commit(‘方法名’)

  • mutations 的函数参数列表中，最多支持两个参数，第一个参数是state状态，第二个参数：通过 commit 提交过来的数据

如果要操作store中的state值，只能通过调用 mutations 提供的方法，才能操作对应的数据，不推荐直接操作 state 数据

  • 因为如果每个组件都能操作数据，万一导致了数据的紊乱，不能快速定位到错误的原因
getters

getters:只负责对外提供数据，不负责修改数据

  • getters与过滤器比较像，在过滤器中也没有修改数据二十八数据做了一层包装，提供给了调用者
  • getters也和computed比较像，只要state中的数据发生变化，getters 正好引用了这个数据，那么就会立即触发getters重新求值
