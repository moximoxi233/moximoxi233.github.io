react基础学习笔记

发表于 2019-10-24 | 分类于 react

React的核心是组件，组件的设计目的是提高代码复用率、降低测试难度和代码复杂度。 React的精髓是函数式编程

react特点
声明式设计 −React采用声明范式，可以轻松描述应用（自动dom操作）。
高效 −React通过对DOM的模拟(虚拟dom)，最大限度地减少与DOM的交互。
灵活 −React可以与已知的库或框架很好地配合。
JSX − JSX 是 JavaScript 语法的扩展。
组件 − 通过 React 构建组件，使得代码更加容易得到复用，能够很好的应用在大项目的开发中。
单向响应的数据流 − React 实现了单向响应的数据流，从而减少了重复代码，这也是它为什么比传统数据绑定更简单。

模块化概念

​ 从代码的角度来分析，把一些可重复使用的代码，抽离为单个模块；便于项目的维护和开发。

组件化概念

​ 从UI界面的角度进行分析，把一些可复用的的UI元素，抽离为单独的组件，便于项目的维护和开发。

组件化的好处：随着项目规模的增大，手中的组件越来越多，很方便能把现有的组件，拼接为一个完整的页面；

虚拟DOM

DOM的本质

​ DOM是浏览器中的概念，用JS对象来表示页面中的元素，并提供了操作DOM对象的API

react中的虚拟DOM

是框架的概念，是开发人员用JS对象来模拟页面上的DOM元素和嵌套关系

  • 本质：用JS对象，来模拟DOM元素和嵌套关系
  • 目的：就是为了实现页面元素的高效更新

Diff算法

different（差异）

  • tree diff

    新旧两个dom树逐层对比的过程，就是Tree Diff；当整棵DOM逐层对比完毕，则所有需要被按需更新的元素，必然会被找到

  • component diff

    在进行 tree diff的时候，每一层中，组件级别的对比，叫做component diff

    • 如果对比之后发现组件类型相同，增认为暂时不需要更新
    • 对比之后发现组件类型不同，则需要移除旧组件，创建新组件，并追加到页面上

  • element diff

    在进行组件对比的时候，如果每个组件类型相同，则需要进行元素级别的对比。这一过程就是element diff

第2种创建组件的方式-class

class es6中

在class花括号内，只能写构造器、静态方法和静态属性、实例方法

使用class关键字来创建组件

  • render 函数作用：渲染当前组件所对应的虚拟DOM元素
  • render函数中，必须 return 合法的jsx虚拟dom结构
  • 在class关键字创建组件中，如果想要使用外界传递过来的props参数，不需接收，直接通过this.props.***来访问

不论是class或者function创建组件，它们的props都是只读的

两种创建组件方式的对比

class关键字创建组件 构造函数创建组件
使用class关键字创建的组件，有自己的私有数据（this.state）和生命周期函数； 使用function创建的组件；只有props，没有自己的私有数据和生命周期函数；
使用class关键字创建的组件；叫做有状态组件 构造函数创建组件，叫做 无状态组件
  • 有状态和无状态组件之间本质区别就是：有无state属性，和有无生命周期函数
    • 如果一个组件需要有自己的私有数据，则推荐使用有状态组件
  • 组中的 props 和 state/data 之间的区别
    • props 中的数据都是外界传递过来的，其数据只可读，不可写
    • state/data,都是组件私有的（通过ajax获取回来的数据，一般都是私有数据），其数据可读可写

修改state中的数据

注意：React中，如果想为state中的数据，重新赋值，不要使用this.state.msg=值，应该调用 React 提供的this.setState({msg:新的值})

this.setState注意点

  • 在 setState 中，只会 把对应的state状态更新，而不会覆盖其他的state状态

  • this.setState方法的执行是异步的。所以想等这个方法执行完后立即拿到最新的值，需要使用this.setState({},callback)

绑定文本框与state中的数据

  • 如果我们只是把文本框的 value属性，绑定到了state 状态，但是，如果不提供 onChange 处理函数，得到的文本框，将只会是一个只读的文本框
  • 当为文本框绑定了一个value值以后，要么同时提供一个readOnly,要么提供一个onChange处理函数
