输入网址后发生了什么

发表于 2019-10-10 | 分类于 web

输入一个URL，Enter之后发生了什么

url解析
  • 地址解析
  • HSTS
  • 检查缓存
DNS解析
  • 浏览器缓存
  • 操作系统缓存
  • 路由器缓存
  • dns缓存
tcp（3次握手）连接
服务器请求处理
  • 接受 TCP 报文后，会对连接进行处理，对HTTP协议进行解析（请求方法、域名、路径等），并且进行一些验证
  • 假如服务器配置了 HTTP 重定向，就会返回一个 301永久重定向响应，浏览器就会根据响应，重新发送 HTTP 请求
浏览器接受响应
  • 浏览器接收到来自服务器的响应资源后，会对资源进行分析
  • 首先查看 Response header，根据不同状态码做不同的事（比如上面提到的重定向）。
    • 如果响应资源进行了压缩（比如 gzip），还需要进行解压。
  • 然后，对响应资源做缓存。
  • 接下来，根据响应资源里的 MIME 类型去解析响应内容（比如 HTML、Image各有不同的解析方式）。
页面渲染

不同的浏览器内核，渲染过程也不完全相同，但大致流程都差不多。

  • IE浏览器 Trident
  • Chrome Blink
  • Safari WebKit
  • 火狐 Gecko

大致流程：

​ HTML 解析—CSS解析—渲染树—布局与绘制—合并渲染层—回流与重绘

  • 回流

    当浏览器发现某个部分发现变化影响了布局时，需要倒回去重新渲染

  • 重绘

    改变了某个元素的背景色、文字颜色等等不会影响周围元素的位置变化时，就会发生重绘。

    • display:none 会触发回流
    • visibility:hidden 只会触发重绘。
