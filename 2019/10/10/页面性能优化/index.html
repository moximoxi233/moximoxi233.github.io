页面性能优化

发表于 2019-10-10 | 分类于 优化方案

在真是项目中由于文件太多，或者相互之间很强依赖性会导致网页加载过慢，那么有什么办法能优化页面性能？

页面性能优化

提升页面性能优化的方法

内容方面

1.减少 HTTP 请求 (Make Fewer HTTP Requests)

2.减少 DOM 元素数量 (Reduce the Number of DOM Elements)

3.使得 Ajax 可缓存 (Make Ajax Cacheable)

针对CSS

1.把 CSS 放到代码页上端 (Put Stylesheets at the Top)

2.从页面中剥离 JavaScript 与 CSS (Make JavaScript and CSS External)

3.精简 CSS (Minify CSS)

4.避免 CSS 表达式 (Avoid CSS Expressions)

针对JavaScript
  1. 脚本放到 HTML 代码页底部 (Put Scripts at the Bottom)
  2. 从页面中剥离 JavaScript 与 CSS (Make JavaScript and CSS External)
  3. 精简 JavaScript (Minify JavaScript)
  4. 移除重复脚本 (Remove Duplicate Scripts)

defer 和 async区别
  • defer： 在html解析完成之后才执行，但有多个时，按照加载顺序执行
  • async： 在加载完之后立即执行，当有多个时，执行顺序与加载顺序无关

利用浏览器缓存

缓存：资源文件（比如图片）在本地的硬盘里存有副本，浏览器下次请求的时候，可能直接从本地磁盘里读取，而不会重新请求图片的url

缓存的分类

  • 强缓存

    • 不用请求服务器，直接使用本地的缓存。
    • 利用 http 响应头中的**Expires或Cache-Control**实现的。

  • 协商缓存

    • 浏览器发现本地有资源的副本，但要先与服务器沟通，才决定是否使用这个副本。

    • 利用http 响应头两对Header

      第一对：Last-Modified、If-Modified-Since

      第二对：ETag、If-None-Match

缓存的原理

缓存是所有性能优化的方式中最重要的一步，这个一定要答好。【重要】

有的人可能会回答local storage 和session storage，其实不是这个。浏览器缓存和存储不是一回事。

使用CDN

浏览器第一次打开页面时，缓存是起不了作用的。使用CDN，效果就很明显。
