ES6学习笔记

发表于 2019-12-04 | 分类于 es6

根据阮一峰博客学习+实际操练

解析赋值

按照一定模式，从数组和对象中提取值，赋值给变量——【解构】

数组解构规则

  1. 当等号两边的“模式匹配”才会被正常赋值

    • 解构不成功，变量的值将会是undefined

    • 如果等号右边“不是数组”也会报错。

      只要具有Iterator接口的数据结构，，都可以采用数组形式的解构赋值

指定默认值
  • ES6内部指定只有值与 undefined 严格相等(===)时，才能使用默认值。
  • 默认值可以使用解构赋值的其他变量但前提是这个变量已被赋值
对象解析赋值

数组解构是按照数组顺序，但是，对象解构是根据属性名。对象解构赋值是先要找到同名属性，再进行赋值

注意，对象解析赋值时，匹配模式问题

  • 对象解构赋值也可以使用默认值，默认值生效的条件就是，属性值全等于 undefined。

字符串扩展

字符串遍历接口

es6为字符串添加了遍历器接口，这样字符串可以被for…of循环遍历。

模板字符串

`hhhhhh${name}hi`

模板字符串中需要使用反引号`,将变量名写到 ${ }. 注：变量一定要声明

字符串中新增的方法

  • String.raw() 用来处理模板字符串，它会将\进行转义，并将所有变量替换

  • 判断一个字符串是否包含另一个字符串

    1. includes() 返回布尔值，–找到目标字符串就返回true
    2. startsWith() 返回布尔值判断字符串是否以某个目标字符串开头
    3. endsWith() 返回布尔值,判断字符串是否以某个目标字符串结尾

  • repeat() 返回一个新的字符串，原字符串重复n次，参数是数值，表示重复几次。

    • 参数不能是负数、Infinity

  • ES2017 引入了字符串补全长度的功能，如果某个字符串不够指定长度，将会在头部/尾部不全

    1. padStart() 用于头部补全，参数1：指定的长度。 参数2：需要补全的字符串

    2. padEnd() 用于尾部补全

  • 消除头部/尾部空格

    1. trimStart() 消除头部空格，返回新的字符串，不会修改原来字符串
    2. trimEnd()

  • matchAll() 返回正则表达式在当前字符串的所有匹配

数值扩展

数值的一些方法

es6中对数值新增了一些方法

  • Number.parseInt(),Number.parseFloat(),将全局方法parseInt()、parseFloat,移植到Number对象上，逐渐减少全局办法，是语言逐渐模块化
  • Number.isInteger() 判断一个数是否为整数，是的话返回true
Math对象的扩展
  • Math.trunc 同于去掉一个数的小数部分，返回整数部分
    • 对于非数值参数，会先使用Number方法先将其转化为数值，再求整数值
    • 对于空直和无法截取到整数的值会返回NaN
  • Math.sigh() 判断一个数是正数、负数、还是0.同样对非数值要先用Number进行转换
    • 返回值：正数—+1 负数—-1 0—0 -0—-0 其他值—NaN
  • Math.cbrt() 会先使用Number方法先将其转化为数值，求立方根

函数扩展

函数参数默认值
  • 使用函数默认值时不能有同名参数
  • 参数全等于undefined时就会使用默认值，传入null没有这个效果

直接将默认值写道参数后面，当并没有传入参数时，使用默认值

与解构赋值默认使用

  • 当设置了对象解构赋值的默认值，且函数参数默认值是一个对象

  • 当没有设置对象解构赋值的默认值，且函数默认值是一个具体属性对象

函数的length属性

  • 当函数参数指定默认值之后，将返回没有指定默认值的参数个数。

    • 当设置的默认值的参数不是尾参数，那么length属性将不会计入后面的参数

  • rest参数不会计入length属性

length属性的含义是，该函数与其传入参数的个数。某个参数指定默认值以后，预期传入的参数个数就包括这个参数了

严格模式

es6中规定，只要函数参数使用了默认值、解构赋值、或者扩展运算符，那么函数内部就不能显示设定为严格模式，否则报错。

箭头函数

()=>{} //书写相当简洁

箭头函数特点

  • 书写简洁，可简化回调函数

  • 箭头函数的this指向

    • 对象不构成单独的作用域，箭头函数定义时作用域指向全局

    • 需要动态this的时候也不应该使用箭头函数

注意：

  • 函数体内的this对象，就是定义时所在的对象，而不是使用时所在的对象
  • 不可以做为构造函数
  • 不可以使用arguments对象
  • 不可以使用yield命令，不能作为Generator函数

数组扩展

扩展运算符

扩展运算符（…），详单与rest的逆运算，将一个数组转为用逗号分隔的参数序列

  • 可以替换apply方法

  • 使用扩展运算符来复制数组，合并数组，与解构赋值结合

  • 将字符串转为真正的数组

  • 可以将一个类似数组的对象转换为一个真正的数组

  • 只要具有Iterator接口的对象，都可以使用扩展运算符，如Map结构，Set结构

Array.from()

用于将两类对象转为真正的数组。

  • 类似数组的对象–dom操作返回的NodeList集合
  • 可遍历的对象

  • 转化为真正的数组

    任何有length属性的对象都可以通过Array.from方法转为数组，这种情况，扩展运算符无法转换。

  • 该函数可以接收第二个参数，类似于数组中map方法，即将数组中的每个元素做相应的操作之后返回。

  • 也可以将字符串转为数组

Array.of()

该方法用于将组值转换为数组，为了弥补构造函数Array()的不足。

  • 该方法总是返回的是参数组成的数组

正则的扩展

Symbol

概述

ES6引入了一种新的原始数据类型Symbol，表示独一无二的值。Symbol值就是通过Symbol函数生成的。Symbol函数可以接受一个参数，表示对Symbol实例的描述。

对象的属性名可以有两种类型：

  • 字符串类型
  • Symbol类型

Symbol值可以转化为布尔值，字符串

Symbol.prototype.description

得到Symbol的描述

作为属性名的Symbol

由于symbol值是独一无二的，所以Symbol值可以作为标识符，用于对象的属性名

注意：Symbol值作为对象属性名时的访问属性值的方式

属性名遍历

作为属性值，遍历对象的时候，该属性不会出现在for…in、for…of循环之中，也不会被Object.keys()、Object.getOwnPropertyNames()、JSON.stringify返回

Object.getOwnPropertySymbols() 可以获取指定对象的所有Symbol属性名，该方法返回的是一个数组

Symbol.for(),Symbol.keyFor()

Symbol.for() 【可以实现重新使用同一个Symbol值】可以接收一个字符串组为参数，然乎搜索有没有以该参数作为名称的Symbol值

  • 如果有，就返回一个Symbol值
  • 否则，新建一个以该字符串为名称的Symbol值

Symbol.keyFor() 该方法返回一个已登记的Symbol类型值的key

Set和Map数据结构

Set

ES6中提供了新的数据结构，它类似于数组，但是其成员的值都是唯一的，没有重复的值

向Set加入值时不会发生类型转换，Set内部会判断两个值是否不同（===），但是向Set加入值时认为NaN等于自身（而NaN在===中是不全等于NaN的）

Set中的属性和方法

  • 属性：

    • Set.prototype.constructor:构造函数默认就是Set函数
    • Set.prototype.size:返回实例成员总数

  • 方法

    • Set.prototype.add(value):添加某个值，返回Set结构本身
    • Set.prototype.delete(value):删除某个值，返回一个布尔值，表示删除是否成功
    • Set.prototype.has(value)：判断该值是否为Set的成员，返回一个布尔值
    • Set.prototype.clear():清除所有成员，无返回值

    Set结构的遍历

    • Set.prototype.keys():返回键名的遍历器

    • Set.prototype.values():返回键值遍历器–keys，values行为完全一致

    • Set.prototype.entries():返回键值对的遍历器

    • Set.prototype.forEach():使用回调函数遍历每个成员

WeakSet

WeakSet与Set类似，也是不重复的值的集合，但是它与Set不同的是，WeakSet成员只能是对象。

WeakSet中没有size属性，也不能遍历

Map

Map是ES6提供的数据结构。它类似于对象，也是键值对的集合，但是“键”的范围不限于字符串，各种类型的值都可以当作键。

  • 对同一个键多次赋值，后面的值会覆盖前面的值

任何具有Iterator接口、且每个成员都是一个双元素的数组的数据结构都可以当作Map勾走函数的参数，Set和Map都可以用来生成新的Map

WeakMap

WeakMap与Map结构类似，也是用于生成键值对的集合

但是与Map有两点区别：

  • WeakMap只接受对象作为键名（null除外），不接受其他类型的值作为键名
  • WeakMap的键名所指向的对象，不计入垃圾回收机制

Proxy

用于修改某些操作的默认行为，即对编程语言进行补充。

相当于,在目标对象之前假设一层拦截，外界对该对象的访问，都必须先通过这层拦截，即可以对外界的访问进行过滤和改写

proxy实际上重载了点运算符，即用自己的定义覆盖了语言原始定义

var proxy=new Proxy(target,handler)

  • 参数1：是所要拦截的对象
  • 参数2：对于每一个被代理的操作，需要提供一个对于的处理函数
Proxy支持的拦截操作
  • get(target,proKey,receiver)——拦截对象属性的读取
  • get( target, proKey, value, receiver)——拦截对象属性的设置
  • has( target,propKey)——拦截propKey inproxy的操作，返回一个布尔值。
  • deleteProperty( target,propKey)——拦截delete proxy[propKey]的操作

Reflect

Reflect对象与Proxy对象一样，都是为了擦偶哦对象而提供的新的API

Reflect的特点

  • 可以从Reflect对象可以那么到语言内部的方法
  • 修改Object上的方法，返回值更合理
  • 让Object操作都变成了函数行为
  • 只要是Proxy对象的方法，就能在Reflect对象上找到对于方法【Reflect对象的方法与Proxy对象的方法一一对应】

Promise对象

Promise的含义

Promise 是一种异步编程的解决方案

相当于是一个容器，里面保存了某个未来才会结束的时间

特点

  • 对象状态不受影响，有三种状态：pending(进行中)、fulfilled(已成功)、rejected(已失败)
  • 一旦状态改变就不会再变。状态改变只有两种可能：
    • pending—>fulfiilled(resolved)
    • pending—>rejected

Iterator和for…of循环

Iterator概念

遍历器(Iterator) 是一种接口，为各种不同的数据结构提供用以的访问机制。任何数据结构，只要部署了Iterator接口，就可以完成遍历操作。

Iterator作用：

  1. 为各种数据结构，提供一个统一的、渐变的数据接口
  2. 是数据结构中的成员按照某种顺序排列
  3. 有了遍历器接口就可以使用for…of方法

Iterator遍历过程：

  1. 创建一个指针对象，指向当前数据结构的起始位置
  2. 第一次调用指针对象的next方法，可以将指针指向数据结构的第一个成员
  3. 第二次调用指针对象的next方法，可以将指针指向数据结构的第二个成员
  4. 不断调用next方法，知道它指向数据结构的结束位置
默认Iterator接口

ES6中默认的Iterator接口部署在数据结构的Symbol.ierator属性

原生具备Iterator接口的数据结构如下：

  • Array
  • Map
  • Set
  • String
  • TypedArray
  • 函数的arguments对象
  • NodeList对象
遍历器对象的return(),throw()

遍历器中除了有next方法，还可以具有return方法和throw方法

  • return方法：
    一般用于提前退出循环，或者在一个对象完成遍历前，清理或释放资源

  • throw方法：

    配合Generator函数使用

for…of循环

ES6引入了 for…of 循环，作为遍历所有数据结构的同意方法。只要是用iterator结构的数据结构，都能使用这个函数进行遍历

for…of 使用范围

包括数组、Set、Map结构、某些类似数组的对象（arguments对象、Dom NodeList对象）、Generator对象、字符串

Generator函数的语法

简介

Generator函数是ES6提供的一种异步编程解决方案。

Generator函数像一个状态机，封装了很多不同的状态在内部，执行Generator函数会返回一个遍历器对象，会一次遍历Generator函数内部的每一个状态。

Generator函数的两个特点：

  • function 关键字与函数名之间有个 *****
  • 函数体内部使用yield表达式，定义不同的内部状态
    • 执行Generator函数会返回一个遍历器对象，可以调用next方法来访问每个状态
next方法的参数

yield表达式本身没有返回值，或者时返回undefined。

next方法可以带一个参数，该参数就会被当作上一个yield表达式的返回值。

async函数

含义

async函数就是Generator函数的语法糖

对比发现async函数中的await就相当于Generator函数中的yield，async就相当于Generator函数中的*。

async函数对Generator函数的改进

  • async自带执行器，不需要像Generator借助next函数来真正执行
  • 更好的语义
  • 更广的适用性
  • 返回值是promise对象

当函数执行时，遇到await就会先返回，等到异步操作完成再继续执行

语法

async函数返回一个Promise对象，async函数内部return语句返回的值，会成为then方法回调函数的参数
